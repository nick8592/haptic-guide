Pipeline

How It Works

Four stages, under 35ms end-to-end. Pan your camera, feel the feedback, find your target.

📹

Camera Capture

~27ms

Low-latency V4L2 capture with MJPG decoding and minimal buffer for real-time frames.

🧠

YOLO26 Detection

4–29ms

End-to-end NMS-free inference via ONNX Runtime on GPU. Zero cloud calls.

🎧

Feedback Engine

~1ms

Maps spatial position to vibration intensity, pitch, beat rate, and stereo pan.

🔊

Audio Output

~3ms

Real-time spatial audio via PipeWire with proximity-driven pitch, rhythm, and earcon.

States

Feedback Modes

Like a metal detector — the closer you get, the stronger the signal. Four distinct states guide you to your target.

IDLE

No Target

No target detected in frame. System waits patiently with no feedback output.

Vibration: None Audio: Off
SCANNING

Far From Center

Target spotted but far from frame center. Slow vibration, low pitch, wide stereo panning.

10Hz · 20% 200Hz · 60 BPM
TRACKING

Getting Closer

Approaching the target. Vibration intensifies, pitch rises, stereo narrows toward center.

40–70Hz · 60–80% 500–600Hz · 180–300 BPM
LOCKED

Target Locked

Target centered. Continuous buzz, sustained earcon — the distinctive "found it" sound.

80Hz · 100% Earcon · Sustained

Setup

Quick Start

Docker-first setup. Build once, run anywhere with GPU passthrough.

Terminal
# Build the Docker image
./scripts/haptic-guide.sh build

# Run with defaults (yolo26n, camera 0, "cell phone")
./scripts/haptic-guide.sh run

# Search for a person
TARGET="person" ./scripts/haptic-guide.sh run

# Enable visual display window
DISPLAY_MODE=on ./scripts/haptic-guide.sh run

# Use a more accurate model
MODEL_VARIANT=yolo26s ./scripts/haptic-guide.sh run
